Etape 3 — Redirect URLs REQUIS

Configure les URLs de redirection pour que l'auth (Google OAuth, email confirm) fonctionne.

  1. Va sur Authentication → URL Configuration ↗
  2. Dans "Site URL", mets :
    https://adventurer-outdoor.vercel.app
  3. Dans "Redirect URLs", clique "Add URL" et ajoute ces 3 URLs :
    https://adventurer-outdoor.vercel.app/auth/callback
    https://adventurer-outdoor.vercel.app
    http://localhost:3000/auth/callback
  4. Clique Save

Etape 4 — Google OAuth OPTIONNEL

Si tu veux activer "Continuer avec Google". Sans ca, le bouton utilisera le mode mock (fallback automatique).

  1. Va sur Google Cloud Console → Credentials ↗
  2. Cree un "OAuth 2.0 Client ID" (type: Web application)
  3. Dans "Authorized redirect URIs", ajoute :
    https://avldyvgvouzpeprygvyw.supabase.co/auth/v1/callback
  4. Copie le Client ID et le Client Secret
  5. Va sur Supabase → Auth → Providers ↗
  6. Active Google, colle le Client ID et le Client Secret, sauvegarde
